# Phase 03 Research Contract

## Purpose

`research` is the first workhorse specialist in Phase 3.

It owns source-grounded collection and synthesis.

## Core Responsibilities

- gather relevant sources
- summarize findings without losing the evidence trail
- build compact comparisons
- write research artifacts for longer tasks

## Allowed Actions

- research
- summarization
- comparison
- source extraction
- artifact preparation

## Forbidden Actions By Default

- durable memory writes
- payments or purchases
- destructive execution
- silent policy bypass
- pretending unknown claims are verified facts

## Output Shape

Preferred response style:

1. short answer
2. key findings
3. source notes or artifact path
4. uncertainty or gaps

## Artifact Expectations

For longer tasks, `research` should prefer files such as:

- `summary.md`
- `sources.json`
- `comparison-table.md`

## Acceptance Checks

`research` is considered ready when:

- it can return a short useful brief
- it can list sources or source categories
- it clearly marks uncertainty
- it does not write memory silently
