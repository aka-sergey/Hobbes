# Канонический комплект документации Hobbes

Этот документ фиксирует 10 главных документов, которые вместе дают самый полезный и актуальный вход в проект.

Статус этого документа:

- `active`

## Зачем нужен канонический комплект

Документации в Hobbes уже много.
Это хорошо для истории проекта, но плохо для быстрого входа.

Поэтому здесь зафиксирован небольшой набор документов, который:

- покрывает всю систему целиком;
- показывает текущее состояние;
- объясняет архитектуру OpenClaw, dashboard и search;
- показывает, где сейчас пробелы;
- не заставляет сразу читать весь rollout-архив.

## Что не вошло в канонический комплект

Не потому что "плохо", а потому что:

- это исторические rollout-артефакты;
- это специальные runbooks;
- это более узкие документы по конкретным подсистемам;
- или это документы, уже частично superseded более новыми обзорными файлами.

## Как пользоваться комплектом

Если нужно быстро войти в проект:

1. прочитать `README`;
2. прочитать `Hobbes_System_Map`;
3. прочитать `Hobbes_Current_State`;
4. прочитать `Hobbes_Gaps_And_Next_Steps`;
5. затем открыть `BLUEPRINT-REVIEW-AND-ROADMAP`;
6. потом углубляться в Telegram, Search, Dashboard и rollout-документы по мере задачи.
