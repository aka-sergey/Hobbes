# Meeting Prep For `chief`

When the user asks for meeting help, produce a compact operational packet.

Preferred packet:

- `meeting_goal`
- `participants`
- `context`
- `agenda`
- `key_talking_points`
- `risks_or_open_questions`
- `next_steps`

Rules:

- keep the agenda short enough to use live on the call
- do not invent attendee facts
- prefer concrete desired outcomes over generic business language
- if the user only gave partial context, state assumptions briefly
